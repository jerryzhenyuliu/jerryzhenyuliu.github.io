Building a neural network from scratch: from one neuron to solving XOR

I wanted to use PyTorch, but before I let a framework do backprop for
me I wanted to make sure I understood what backprop is. So I wrote a
neural network in pure Python — no NumPy, no PyTorch — and built it up
from the smallest possible piece. One neuron. Then two. Then a whole
layer. Then a network.

This is the write-up. The surprise at the end: the whole thing, the
part that people call “how neural networks learn”, is just the chain
rule applied sensibly. Nothing hidden. Nothing magic.

Step 1: one neuron

A neuron is literally two lines of code:

Given weights $(w_1, w_2, b)$ you can hand-tune a neuron to separate
two clouds of 2D points. Here’s what I got trying different weights on
blue-around-$(-2,-2)$ versus red-around-$(2,2)$:

$w_1$ $w_2$ $b$ accuracy
1.0 1.0 0.0 100.0%
1.0 1.0 -2.0 99.0%
0.5 0.5 0.0 100.0%
-1.0 -1.0 0.0 0.0% (flipped)

For two cleanly-separated clouds, a human can find good weights by
guessing. Three dials is tractable. But by the time you have 17
parameters (a 2-4-1 network), there’s no way. You need the computer
to find them.

Step 2: gradient descent

Loss = mean squared error between the neuron’s output and the true
label. You compute the gradient of the loss with respect to each
parameter, then nudge each parameter in the direction that reduces
loss. Repeat for many epochs.

The gradients are just derivatives, applied layer by layer:

Three things to notice:

  1. d_sigmoid = pred * (1 - pred) — the derivative of sigmoid
    conveniently uses the output of the neuron, not the input.
    That’s a standard trick that saves recomputation.
  2. Each line is the chain rule: dL/dwi = (dL/dpred) * (dpred/dz) * (dz/dwi).
    The only “interesting” part is (dz/dwi) = xi.
  3. This code alone, with one neuron and no hidden layer, already finds
    100% accurate weights for the two-cloud problem.

Step 3: adding a hidden layer (and deriving backprop)

To solve hard classification problems — ones where a single straight
line can’t separate the classes — I need a hidden layer. The classic
example is XOR:

You cannot draw a line that separates the 1s from the 0s.

The 2-layer network looks like:

with two weight matrices, $W_1$ and $W_2$. To train it, I need
$\partial L / \partial W_1$ and $\partial L / \partial W_2$. The chain
rule, spelled out:

$$
\begin{aligned}
\frac{\partial L}{\partial \hat{y}} &= 2(\hat{y} - y) \
\frac{\partial \hat{y}}{\partial z_{\text{out}}} &= \hat{y}(1 - \hat{y}) \
\frac{\partial z_{\text{out}}}{\partial W_2} &= h \
\frac{\partial z_{\text{out}}}{\partial h} &= W_2 \
\frac{\partial h}{\partial z_{\text{hidden}}} &= h(1 - h) \
\frac{\partial z_{\text{hidden}}}{\partial W_1} &= x
\end{aligned}
$$

Multiply these together in the right order (called the backward
pass) and you get the gradient for every parameter. That’s
backpropagation. That is the entirety of “how neural networks learn”.

Step 4: verify the derivation before trusting it

Before I trained anything, I checked my gradients numerically.

For any parameter $w$:

$$
\frac{\partial L}{\partial w} \approx \frac{L(w + \epsilon) - L(w - \epsilon)}{2\epsilon}
$$

So I can run forward() twice with a tiny perturbation, divide, and
compare against the gradient my backward pass computed. Here’s the
output for the first few weights:

Relative error under $10^{-7}$. The analytic gradient matches the
finite-difference gradient to about ten digits. Backprop is implemented
correctly.

This is the single most valuable habit I picked up from the project.
Whenever I implement a gradient anywhere, I gradient-check it on a
small example. Bugs in gradient code are silent — they look like “the
model isn’t learning well” — and gradient-check catches them instantly.

Step 5: solve XOR

2-4-1 network, sigmoid everywhere, lr = 2.0, 10,000 epochs. Training
loss:

Three different hidden-layer sizes. Things to notice:

  • 2-2-1 plateaus for about 500 epochs before dropping. That’s a
    “symmetry-breaking” period: with only 2 hidden neurons, the network
    has to escape a local-minimum-ish region before it finds a layout that
    works.
  • 2-4-1 and 2-8-1 drop right away. More hidden neurons means more
    ways to stumble onto a good configuration.
  • All three reach ~$10^{-4}$ loss eventually, but 2-8-1 does it fastest
    and most smoothly.

Final predictions (2-4-1):

input target network output
(0, 0) 0 0.0033
(0, 1) 1 0.9921
(1, 0) 1 0.9923
(1, 1) 0 0.0117

XOR solved. Better than perfect — the network isn’t just right, it’s
confident.

What the hidden neurons actually learned

Looking at the trained weights of the 2-4-1 network:

Each hidden neuron learned a line in the (x1, x2) plane. Four lines.
The output layer then combines them nonlinearly — “big if hidden[0] is
big AND hidden[1] is small AND …” etc — to carve out the XOR pattern.

This is why a single neuron can’t solve XOR but a 2-layer network can:
one line can’t separate XOR’s 1s from its 0s, but two lines composed by
a second layer absolutely can. Minsky and Papert proved in 1969 that a
single-layer perceptron is stuck, which nearly killed neural-network
research for a decade. The fix — stacking layers — sounds obvious in
retrospect; the hard part was getting backpropagation right so you
could actually train the deeper networks.

A harder problem: concentric rings

To make sure the NN class generalized, I threw a harder 2D problem at
it — points arranged in two concentric rings, inner radius ~1 (label 0)
and outer radius ~3 (label 1). No straight line separates them, and
even two lines can’t.

A 2-8-1 network with sigmoid activations learns a curved decision
boundary — a ring of #s surrounding a central empty region. That’s a
nonlinear shape emerging from a pile of linear pieces.

What I’d change next

  • ReLU instead of sigmoid. Sigmoid saturates: far from zero its
    derivative is close to zero, so gradients shrink as you go deeper.
    ReLU (max(0, x)) has derivative 1 wherever it’s active, which is
    why every modern deep network uses it.
  • Cross-entropy loss for classification. Gives bigger gradients
    when the model is badly wrong.
  • NumPy. My loops are Python-level. Vectorizing the forward and
    backward pass with NumPy would be a ~100x speedup for free, without
    changing any of the math.
  • A real dataset. Still 2D input in my experiments; next step is
    MNIST (784-dimensional input, 10-class output). My pure-Python
    implementation would be way too slow, but that’s what NumPy is for.

The takeaway

Before this project, “the network learns by backpropagation” was a
black box to me. After writing every line by hand and watching the
gradient check pass, it’s… not a black box. It’s a for-loop.

PyTorch is still amazing — it does this for you, it does it in
CUDA, it handles arbitrary computation graphs. But now I know what
it’s doing. Which means when it breaks, I’ll have some hope of
debugging it.
